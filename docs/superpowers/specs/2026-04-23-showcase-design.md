**Date:** 2026-04-23
**Repo:** tku-mackay
**Deployment:** tku-mackay.vercel.app

## 背景

馬偕好厝邊松年大學 114 下學期「人工智慧（AI）生活應用」課程即將結業。課程主軸：讓學員用 AI 工具把人生故事寫成一首歌（透過 Gemini/ChatGPT 採訪 → 歌詞 → Suno 生成）。

課程結束，需要把首頁從「課程講義索引」轉成「作品成果展」，讓學員可以把專屬頁面分享給家人朋友。

## 核心決策

### 定位轉向
- 首頁從「課程導覽」轉成「成果展首頁」，以作品優先
- 講義摺疊收進底部 `<details>` 區塊，學員回顧時展開

### 敘事結構
- 老師（Powei）創作一首「班級主題曲」當 Opener，講與學員的緣分
- 結構：1 首老師的歌 + 6 首學員的歌 = 7 首「班級畢業專輯」
- 老師的歌先放 placeholder（學員作品到位後再寫）

### 用詞
- 一律用「學員」，不用「長輩」
  （參與者為學習主體，不以年齡定義）

### 視覺延用
- 沿用現有色系：磚紅 `#B5411F`、米白背景 `#FAFAF8`、暖邊框 `#C09A6E`
- Font：`Noto Sans TC` + `Noto Serif TC`（沿用 `ahui.html` 搭配）

## 學員個人頁樣板

每位學員一份獨立 HTML 檔（不抽共用 template），但 **CSS 抽出到共用檔 `stories/stories.css`**（7 頁 CSS 重複度約 95%，只差 hero 漸層色；抽檔可避免改一次要改 7 處）。

每頁 HTML 只保留結構 + 個人色 tokens（透過 `<body style="--hero-from:...; --hero-to:...">` 或小 `<style>` 區塊注入）。以 `stories/ahui.html` 的結構為基礎，每頁結構：

```
┌──────────────────────────────────┐
│ ← 回成果展                        │
├──────────────────────────────────┤
│ Hero（漸層背景）                  │
│   課程 badge                      │
│   《歌曲標題》                    │
│   學員名                          │
├──────────────────────────────────┤
│ 專輯封面（大圖，方形）            │
├──────────────────────────────────┤
│ 🎵 「在 Suno 上聆聽」按鈕         │
├──────────────────────────────────┤
│ 歌詞                              │
│   [Verse] 小標                    │
│   全形空白保留                    │
│   [Chorus] 小標                   │
├──────────────────────────────────┤
│ 📖 歌曲背後的故事                 │
│   （散文版，段落清楚）            │
├──────────────────────────────────┤
│ 學員介紹                          │
│   頭銜 + 人生觀                   │
├──────────────────────────────────┤
│ ← 回成果展                        │
└──────────────────────────────────┘
```

### 關鍵設計重點

**兩層文字**：歌詞（詩的版本）和 故事（散文版本）分開展示，讓訪客可以邊聽歌邊讀歌詞，然後往下讀完整故事。

**歌詞格式處理**：
- `[Verse]` `[Chorus]` `[Bridge]` `[Intro]` `[Outro]` 等標記 → 轉成小標樣式（不同顏色/字重）
- 全形空白 `　` → 保留渲染（讓詩句節奏呼吸）
- 換行忠實呈現

**故事（散文）處理**：
- CSV「簡介」欄位經常混了「短自介 + 長故事」→ 拆成兩段
- 清除 Markdown 符號（`###`、`---`），但保留 emoji 和語意結構
- 空白行當段落分隔，渲染成 `<p>`

**封面圖**：
- 檔案放 `stories/covers/<slug>.jpg`
- 找不到檔案時 fallback CSS 漸層 + 歌曲標題首字（或 emoji）
- 尺寸建議 800×800

**Suno 連結按鈕**（與 `ahui.html` 一致）：
不做 iframe embed（避免 URL 格式差異 + CSP 風險）。每頁放一個大按鈕「🎵 在 Suno 上聆聽」，`href` 直接是學員的 Suno URL。
- 有效 Suno URL（任何格式 `suno.com/s/...` 或 `suno.com/song/...`）→ 按鈕連過去
- 阿勃勒的 Gemini 連結 → 改成「🎵 歌曲製作中」灰階按鈕，`disabled` 狀態
- 空連結 → 同上

**Placeholder 策略**：
- 歌詞缺（Johnson、阿勃勒）→ 顯示「🎼 歌詞整理中，敬請期待」
- Suno 缺（阿勃勒）→ 顯示「🎵 歌曲製作中」卡片
- 封面缺（全部初期都缺）→ CSS 漸層 + 首字

**導航**：
- 頁首＋頁尾都有「← 回成果展」
- 不做「上一位／下一位」（不想要制式化，保留自由探索）

## 資料處理細節

### 清理規則（全部學員）
- `.strip()` 所有欄位（名字、頭銜、Suno URL）
- 歌詞保留原始換行、全形空白、`[Section]` 標記
- 故事清除 `###` `---`；保留 emoji

## 部署

- 現有 Vercel deployment（`tku-mackay.vercel.app`）
- 純靜態 HTML，push 到 main 自動部署
- 無 build step

## 上線流程

1. 實作所有頁面與首頁改版，所有資料缺口以 placeholder 填入
2. Local preview（`open index.html`）／Vercel preview deploy 檢查
3. Push 上線
4. 老師（Powei）把連結發群組，請學員檢查自己的頁面、補資料、補封面圖
5. 老師收到學員回覆後，陸續補進對應頁面
6. 學員作品全部到位後，老師寫「班級主題曲」，補進 `teacher.html`

## 範圍外（不做）

- 不做後端／資料庫（純靜態足夠 6 位學員）
- 不做上/下一位學員導航（維持探索感）
- 不整合原 `hosted.pwlee.xyz`（那是開學 demo，功能已被此成果展取代）
- 不做多語系（觀眾群是中文圈家人朋友）
- 不做留言／互動（不是主要需求，真有需要再說）
